# Sorry-solving cheat sheet (Tao Analysis in Lean)

Working notes for solving `sorry`s in `analysis/Analysis/*.lean`.

## Workflow
- Build a single file fast: `lake env lean Analysis/Section_X_Y.lean` (no `lake build` needed for a quick check). Exit 0 + no `declaration uses 'sorry'` warning = clean.
- Find easy targets: `grep -rc sorry --include=*.lean Analysis | grep -v :0 | sort -t: -k2 -n`.
- Commit after each net decrease in sorrys: `git commit -m "solve N sorrys in Section_X_Y.lean"`. End message with the Co-Authored-By trailer.
- `-Dwarn.sorry=false` is set for `lake build`, but `lake env lean <file>` DOES print sorry warnings тАФ use that to confirm.

## Tips learned
- **Instance fields**: when filling an `instance ... where` field, `#print <ClassName>` to get the EXACT field signature and binder order. The `intro` order must match the field's argument order, NOT the standalone lemma's. Example: `MulPosMono.mul_le_mul_of_nonneg_right : тИА тжГcтжД, 0 тЙд c тЖТ тИА тжГa bтжД, a тЙд b тЖТ a*c тЙд b*c` тЖТ `intro c _ a b hab`.
- The Chapter 2 `Nat` defines `тЙд` as `тИГ d, b = a + d`; `obtain тЯиd, hdтЯй := hab` then `rw [hd, add_mul/mul_add]`.
- Mirror a sibling proof in the same file when one direction (left/right) is already done.
- **Restricted-filter limits** `atTop тКУ ЁЭУЯ (range f)`: rewrite with `тЖР Filter.map_comap` (gives `map f (comap f atTop) = atTop тКУ ЁЭУЯ (range f)`), then `Nat.comap_cast_atTop` (`comap cast atTop = atTop`) collapses it to `map cast atTop`, then `Filter.tendsto_map'_iff` turns `Tendsto g (map f l)` into `Tendsto (gтИШf) l`. Great for "sequence vs function-on-тДЭ-restricted-to-тДХ" equivalences.
- `tendsto_inv_atTop_zero : Tendsto (┬╖тБ╗┬╣) atTop (nhds 0)`; combine with `Filter.Tendsto.mono_left _ inf_le_left` for restricted filters; `simpa [one_div]` to match `1/x`.
- **IVT / continuous image of Icc**: `ContinuousOn.image_Icc (hab : a тЙд b) : f '' Icc a b = Icc (sInf ..) (sSup ..)`. `intermediate_value_Icc (hab) (hf) : Icc (f a) (f b) тКЖ f '' Icc a b`; primed version `intermediate_value_Icc'` uses `Icc (f b) (f a)` (decreasing). Fixed-point: set `g = f x - x`, apply `intermediate_value_Icc'` to get `0 тИИ g '' Icc`.
- **Monotone deriv тЙе 0**: `hderiv.hasDerivWithinAt` then `rw [hasDerivWithinAt_iff_tendsto_slope]`; bound `slope f xтВА y тЙе 0` via `slope_def_field` + `div_nonneg`/`div_nonneg_iff` (no `div_nonneg_of_nonpos_of_nonpos` lemma тАФ use `div_nonneg_iff` and pick `Or.inr`); finish with `ge_of_tendsto` (need `NeBot` from the `ClusterPt` hyp: `haveI : (nhdsWithin xтВА (X\{xтВА})).NeBot := hxтВА`).
- **derivWithin = deriv** on interior of Icc: `rw [тЖР derivWithin_of_mem_nhds (Icc_mem_nhds hx.1 hx.2)]` (after `rw [interior_Icc]`). Or `hf.derivWithin huniq` / `hderiv.hasDerivWithinAt.derivWithin huniq` where `huniq : UniqueDiffWithinAt`.
- **StrictMonoOn from positive deriv**: `strictMonoOn_of_deriv_pos (convex_Icc a b) hf.continuousOn`; the `hf'` arg needs `deriv` (global) on `interior D` тАФ convert from `derivWithin`.
- **UniqueDiffWithinAt for a set union**: `(uniqueDiffOn_Icc (by norm_num) x hmem).mono Set.subset_union_left/right`. `UniqueDiffWithinAt.mono` is monotone in the set (s тКЖ t).
- **Counterexample to "positive derivWithin тЯ╣ strict mono"** needs a DISCONNECTED domain (the theorem holds on intervals). Use `f = x┬│-3x` on `[-2,-3/2] тИк [3/2,2]`: globally differentiable (poly), `f'=3x┬▓-3>0` since `|x|тЙе3/2`, but `f(-3/2)=9/8 > f(3/2)=-9/8`. - `Set.mem_image_of_mem` (NOT bare `mem_image_of_mem`). `Set.mem_Icc` to unpack `x тИИ Icc a b` into `a тЙд x тИз x тЙд b`.
- **Beta reduction is finicky**: after `refine тЯиfun x => тАж, тАжтЯй`, sometimes the goal/hyp is auto-beta-reduced and `dsimp only` then ERRORS with "no progress"; other times it's NOT reduced and `split_ifs`/`rw` fails to match the lambda. Don't blanket-add `dsimp only`. For a goal use `show <reduced form>` to force it; for hyps from `h x hx`, try WITHOUT dsimp first (often already reduced) and only add `dsimp only at h` if a later `rw` fails to match. `nlinarith` ring-normalizes so it tolerates unreduced lambdas.
- **`IsMinOn`/`IsMaxOn`**: `rw [isMinOn_iff]`/`isMaxOn_iff` тЖТ `тИА x тИИ s, f a тЙд f x` / `тИА x тИИ s, f x тЙд f a`. To refute existence of a max on an OPEN interval, pick a witness strictly past the candidate toward the (excluded) endpoint, e.g. `3/2 + (|xтВА-3/2|+1/2)/2`, and `nlinarith` with `sq_abs`, `abs_nonneg`, and an explicit `mul_pos` of the two positive factors.
- `max M 0 + 1` is the canonical "bigger than M and тЙе 1" gadget for unboundedness proofs; `le_max_left`/`le_max_right` give the two bounds (mind which!). `1/(max M 0+1)` lands in `(0,1]`.
