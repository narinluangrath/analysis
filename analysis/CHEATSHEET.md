# Sorry-solving cheat sheet (Tao Analysis in Lean)

Working notes for solving `sorry`s in `analysis/Analysis/*.lean`.

## Workflow
- Build a single file fast: `lake env lean Analysis/Section_X_Y.lean` (no `lake build` needed for a quick check). Exit 0 + no `declaration uses 'sorry'` warning = clean.
- Find easy targets: `grep -rc sorry --include=*.lean Analysis | grep -v :0 | sort -t: -k2 -n`.
- Commit after each net decrease in sorrys: `git commit -m "solve N sorrys in Section_X_Y.lean"`. End message with the Co-Authored-By trailer.
- `-Dwarn.sorry=false` is set for `lake build`, but `lake env lean <file>` DOES print sorry warnings ‚Äî use that to confirm.

## Tips learned
- **Instance fields**: when filling an `instance ... where` field, `#print <ClassName>` to get the EXACT field signature and binder order. The `intro` order must match the field's argument order, NOT the standalone lemma's. Example: `MulPosMono.mul_le_mul_of_nonneg_right : ‚àÄ ‚¶Éc‚¶Ñ, 0 ‚â§ c ‚Üí ‚àÄ ‚¶Éa b‚¶Ñ, a ‚â§ b ‚Üí a*c ‚â§ b*c` ‚Üí `intro c _ a b hab`.
- The Chapter 2 `Nat` defines `‚â§` as `‚àÉ d, b = a + d`; `obtain ‚ü®d, hd‚ü© := hab` then `rw [hd, add_mul/mul_add]`.
- Mirror a sibling proof in the same file when one direction (left/right) is already done.
- **Restricted-filter limits** `atTop ‚äì ùìü (range f)`: rewrite with `‚Üê Filter.map_comap` (gives `map f (comap f atTop) = atTop ‚äì ùìü (range f)`), then `Nat.comap_cast_atTop` (`comap cast atTop = atTop`) collapses it to `map cast atTop`, then `Filter.tendsto_map'_iff` turns `Tendsto g (map f l)` into `Tendsto (g‚àòf) l`. Great for "sequence vs function-on-‚Ñù-restricted-to-‚Ñï" equivalences.
- `tendsto_inv_atTop_zero : Tendsto (¬∑‚Åª¬π) atTop (nhds 0)`; combine with `Filter.Tendsto.mono_left _ inf_le_left` for restricted filters; `simpa [one_div]` to match `1/x`.
- **IVT / continuous image of Icc**: `ContinuousOn.image_Icc (hab : a ‚â§ b) : f '' Icc a b = Icc (sInf ..) (sSup ..)`. `intermediate_value_Icc (hab) (hf) : Icc (f a) (f b) ‚äÜ f '' Icc a b`; primed version `intermediate_value_Icc'` uses `Icc (f b) (f a)` (decreasing). Fixed-point: set `g = f x - x`, apply `intermediate_value_Icc'` to get `0 ‚àà g '' Icc`.
- **Monotone deriv ‚â• 0**: `hderiv.hasDerivWithinAt` then `rw [hasDerivWithinAt_iff_tendsto_slope]`; bound `slope f x‚ÇÄ y ‚â• 0` via `slope_def_field` + `div_nonneg`/`div_nonneg_iff` (no `div_nonneg_of_nonpos_of_nonpos` lemma ‚Äî use `div_nonneg_iff` and pick `Or.inr`); finish with `ge_of_tendsto` (need `NeBot` from the `ClusterPt` hyp: `haveI : (nhdsWithin x‚ÇÄ (X\{x‚ÇÄ})).NeBot := hx‚ÇÄ`).
- **derivWithin = deriv** on interior of Icc: `rw [‚Üê derivWithin_of_mem_nhds (Icc_mem_nhds hx.1 hx.2)]` (after `rw [interior_Icc]`). Or `hf.derivWithin huniq` / `hderiv.hasDerivWithinAt.derivWithin huniq` where `huniq : UniqueDiffWithinAt`.
- **StrictMonoOn from positive deriv**: `strictMonoOn_of_deriv_pos (convex_Icc a b) hf.continuousOn`; the `hf'` arg needs `deriv` (global) on `interior D` ‚Äî convert from `derivWithin`.
- **UniqueDiffWithinAt for a set union**: `(uniqueDiffOn_Icc (by norm_num) x hmem).mono Set.subset_union_left/right`. `UniqueDiffWithinAt.mono` is monotone in the set (s ‚äÜ t).
- **Counterexample to "positive derivWithin ‚üπ strict mono"** needs a DISCONNECTED domain (the theorem holds on intervals). Use `f = x¬≥-3x` on `[-2,-3/2] ‚à™ [3/2,2]`: globally differentiable (poly), `f'=3x¬≤-3>0` since `|x|‚â•3/2`, but `f(-3/2)=9/8 > f(3/2)=-9/8`. Build deriv via `HasDerivAt`: `simpa using hasDerivAt_pow 3 x`, `(hasDerivAt_id x).const_mul 3`, `.sub`.
- `Set.mem_image_of_mem` (NOT bare `mem_image_of_mem`). `Set.mem_Icc` to unpack `x ‚àà Icc a b` into `a ‚â§ x ‚àß x ‚â§ b`.
- **Beta reduction is finicky**: after `refine ‚ü®fun x => ‚Ä¶, ‚Ä¶‚ü©`, sometimes the goal/hyp is auto-beta-reduced and `dsimp only` then ERRORS with "no progress"; other times it's NOT reduced and `split_ifs`/`rw` fails to match the lambda. Don't blanket-add `dsimp only`. For a goal use `show <reduced form>` to force it; for hyps from `h x hx`, try WITHOUT dsimp first (often already reduced) and only add `dsimp only at h` if a later `rw` fails to match. `nlinarith` ring-normalizes so it tolerates unreduced lambdas.
- **`IsMinOn`/`IsMaxOn`**: `rw [isMinOn_iff]`/`isMaxOn_iff` ‚Üí `‚àÄ x ‚àà s, f a ‚â§ f x` / `‚àÄ x ‚àà s, f x ‚â§ f a`. To refute existence of a max on an OPEN interval, pick a witness strictly past the candidate toward the (excluded) endpoint, e.g. `3/2 + (|x‚ÇÄ-3/2|+1/2)/2`, and `nlinarith` with `sq_abs`, `abs_nonneg`, and an explicit `mul_pos` of the two positive factors.
- `max M 0 + 1` is the canonical "bigger than M and ‚â• 1" gadget for unboundedness proofs; `le_max_left`/`le_max_right` give the two bounds (mind which!). `1/(max M 0+1)` lands in `(0,1]`.
- **Chapter 4 rationals (`Section_4_2.Rat`, quotient of `a // b` with `b‚âÝ0`)**: destructure `obtain ‚ü®a,b,hb,rfl‚ü© := eq_diff x` (carries the `b‚âÝ0` proof). `Rat.eq a c hb hd : a//b = c//d ‚Üî a*d = c*b` (needs BOTH denom‚âÝ0 proofs). `add_eq`/`mul_eq`/`neg_eq`/`inv_eq` similarly take denom-nonzero args. Ring fields: obtain all, `rw [add_eq/mul_eq ‚Ä¶, eq _ _ <Int.mul_ne_zero‚Ä¶> <‚Ä¶>]; ring` ‚Äî supply each composite denominator's nonzero proof via `Int.mul_ne_zero`/`one_ne_zero`. Setoid `trans`: `linear_combination f*h1 + b*h2` then `mul_right_cancel‚ÇÄ hd`. Well-defined lifts: `simp_all [Quotient.eq]` often closes, else `linear_combination`. `inv` lift: `by_cases a=0` (junk-value branch closed by `simp [formalDiv, ‚Ä¶]`; nonzero branch by `Rat.eq` + `linear_combination`). `decidableEq`: `apply decidable_of_iff (a*d=c*b); constructor <;> [Quotient.sound; Quotient.exact]` (‚âà is defeq to the ‚Ñ§ equation). `Field.mul_inv_cancel`: derive numerator‚âÝ0, then `inv_eq, mul_eq, eq; ring`. intCast/RingHom maps: `(intCast_add a b).symm` etc. Cast injectivity needs a beta-reducing `have h2 : (‚Üëa:Rat) = ‚Üëb := h` before rewriting. Bare `example`s outside the `Rat` namespace need `Rat.` prefixes (e.g. `Rat.div_eq`).
- **Chapter 4 integers (`Section_4_1.Int`, quotient of formal diffs `a ‚Äî‚Äî b`)**: destructure any `x:Int` with `obtain ‚ü®a,b,rfl‚ü© := eq_diff x`. Key rewrites: `add_eq`/`mul_eq`/`neg_eq` (compute on representatives, defeq via `Quotient.lift_mk`), `Int.eq` (`a‚Äî‚Äîb = c‚Äî‚Äîd ‚Üî a+d = c+b`), `natCast_eq` (`‚Üën = n‚Äî‚Äî0`), and `(0:Int) = 0‚Äî‚Äî0` / `(1:Int) = 1‚Äî‚Äî0` / `(-1:Int) = 0‚Äî‚Äî1` by `rfl`. Ring-instance fields: `obtain` all vars, `simp_rw [add_eq, mul_eq]; congr 1 <;> ring`; `AddGroup.ofLeftAxioms` takes (assoc)(zero_add)(neg_add). Order: `le_iff` (`a‚â§b ‚Üî ‚àÉt:‚Ñï, b=a+‚Üët`), `lt_iff`, `lt_iff_exists_positive_difference`. To kill `a = a + ‚Üëx` use `add_left_cancel` on `a+0 = a+‚Üëx` (NOT `self_eq_add_right` ‚Äî doesn't exist); when `a` appears on BOTH sides of a rw target, use `push_cast; linear_combination ‚Ä¶` instead of `rw`. Cross-type goals (‚Ñï casts in ‚Ñ§) close with `omega`/`push_cast; omega`/`linear_combination`. No-zero-divisors: expand `(a‚Äî‚Äîb)(c‚Äî‚Äîd)`, get `((p:‚Ñ§)-q)*((r:‚Ñ§)-s)=0`, apply `_root_.mul_eq_zero`. Build `‚âÉ+*o` to Mathlib `‚Ñ§` via `Quotient.lift (fun ‚ü®a,b‚ü© => (a:‚Ñ§)-b)`, inverse `z ‚Ü¶ z.toNat ‚Äî‚Äî (-z).toNat`; `left_inv`/`right_inv`/`map_*` all close by `show <reduced ‚Ñ§ form>` + `omega`/`push_cast; ring`. CAUTION: a lemma can only transport through the iso if it's stated AFTER the iso is defined ‚Äî early lemmas (mul_eq_zero, cancel, mul_lt) must be proved directly.
- **Chapter 2 Peano (`Chapter2.Nat`, inductive zero/succ, `n++` = succ)**: `Nat.induction (motive) (base:P 0) (step:‚àÄn, P n‚ÜíP(n++)) : ‚àÄn, P n` is the recursor ‚Äî use `refine Nat.induction (fun n => ‚Ä¶) ?_ ?_ x` (NOT Mathlib `induction`); beta-reduce motive goals with `show` before `rw`. Core rewrites: `succ_add` (`n++ + m = (n+m)++`, rfl), `add_succ`, `zero_add`/`add_zero`, `one_add` (`1+m = m++`). `1 = 0++` by rfl. Order: `Nat.le_iff (n m) : n‚â§m ‚Üî ‚àÉa, m=n+a` (TAKES explicit args ‚Äî `(le_iff m N).mpr`, not `le_iff.mpr`); `lt_iff`, `lt_iff_succ_le` (`a<b ‚Üî a++‚â§b`), `le_iff_lt_or_eq`. CAUTION: Mathlib `add_right_cancel` needs `IsRightCancelAdd` (absent) ‚Äî use the local `Nat.add_left_cancel a b c : a+b=a+c ‚Üí b=c` (after `add_comm`). `abel` works once `AddCommMonoid` instance is in scope (defined mid-file). `rcases d with _ | d'` gives the `zero` constructor (NOT syntactic `0`) ‚Äî `rw [add_zero]` won't match `a + zero`; use defeq via `hd.trans (add_zero a)` or `simp`. Predecessor of a positive: `rcases a with _ | b` then `absurd rfl ha`. Strong/backwards/from-point induction: build a `key : ‚àÄ N, ‚Ä¶` by `Nat.induction` on the bound, then instantiate (e.g. `key (m++) m hm (succ_gt_self m)`).
- **Chapter 2 epilogue (Chapter2.Nat ‚Üî ‚Ñï)**: `toNat` conversion; `zero_toNat`/`succ_toNat` (`(n++).toNat = n.toNat+1`). Preserve `+`/`*`/`^` by `induction' n with n hn` then `rw [succ_add/succ_mul/pow_succ, succ_toNat, hn, ‚Ä¶]` and finish `omega`/`ring` (qualify Mathlib lemmas with `_root_.`, e.g. `_root_.Nat.zero_add`, `_root_.pow_succ`). Order: `Nat.le_iff : n‚â§m ‚Üî ‚àÉa, m=n+a` vs Mathlib `le_iff_exists_add`; bridge via `equivNat.injective` and `equivNat.right_inv : (‚Üëk:Chapter2.Nat).toNat = k`.
- **Abstract Peano (`PeanoAxioms`)**: `natCast P : ‚Ñï ‚Üí P.Nat` (recursive). Injective: auxiliary `‚àÄ n m, natCast n = natCast m ‚Üí n = m` by `induction' n` + `cases m` (cross cases via `P.succ_ne`, succ case via `P.succ_cancel` + `congrArg`). Surjective: `refine P.induction (fun y => ‚àÉ n, natCast n = y) ‚ü®0, rfl‚ü© ?_ y`. Build `P.Nat ‚âÉ ‚Ñï` with `Equiv.ofBijective P.natCast ‚ü®inj, surj‚ü©` (NOT `Function.invFun` ‚Äî needs `Nonempty`; use `Function.surjInv`/`leftInverse_surjInv`/`rightInverse_surjInv` if hand-rolling). `natCast 0 = P.zero` and `natCast (m+1) = P.succ (natCast m)` hold by `rfl`. Transport recursion (`recurse_uniq`): define `g : ‚Ñï ‚Üí P.Nat` by `Nat.rec`, set `a x := g (ic.symm x)`; key `ic.symm (P.succ x) = ic.symm x + 1` via `ic.injective` + `apply_symm_apply` + the succ-rfl. `Equiv` (class) `equiv_zero`/`equiv_succ` for symm/trans: `apply equiv.injective; simp only [equiv_succ, Equiv.apply_symm_apply]` / `simp only [Equiv.trans_apply, ‚Ä¶]`. P.induction goals need `show` to beta-reduce the motive before `rw`.
- **Cardinality (Chapter 8)**: `EqualCard X Y := ‚àÉ f, Bijective f`; `LeCard X Y := ‚àÉ f, Injective f`; `LtCard X Y := LeCard X Y ‚àß ¬¨ EqualCard X Y`. Schr√∂der‚ÄìBernstein = `Function.Embedding.schroeder_bernstein (hf:Injective f) (hg:Injective g) : ‚àÉ h, Bijective h` ‚Äî exactly `LeCard X Y ‚Üí LeCard Y X ‚Üí EqualCard X Y`. From `EqualCard X Y` (a bijection `e`) get the reverse injection via `(Equiv.ofBijective e he).symm` + `.injective`. For `LtCard` transitivity/antisymmetry, reduce `¬¨EqualCard`/`¬¨LeCard` goals to a contradiction by building both injections and applying Schr√∂der‚ÄìBernstein. `Preorder.lt_iff_le_not_ge` field goal is `a<b ‚Üî a‚â§b ‚àß ¬¨b‚â§a`.
- **Chapter 3 SetTheory.Set algebra**: `apply ext; intro x` reduces set equality to membership iff (`SetTheory.Set.ext` from `‚àÄx, x‚ààX ‚Üî x‚ààY`). Membership lemmas all take explicit Object/Set args: `mem_inter x A B`, `mem_union x A B`, `mem_sdiff` (`x‚ààA\B ‚Üî x‚ààA ‚àß x‚àâB`), `mem_singleton x a`, `mem_pair x a b`. For partition/union-decomposition identities, `simp only [mem_union, mem_inter, mem_sdiff]` then `by_cases hA : x‚ààA <;> by_cases hB : x‚ààB <;> tauto`. `eq_empty_iff_forall_notMem` + `not_mem_empty` for emptiness contradictions. Coercion `(X:Object)` is injective: `coe_eq_iff`/`coe_eq` (and `Subtype.coe_injective` for element subtypes). `{a,b}={c,d}` decomposition (`pair_eq_pair`): extract the 4 membership facts via `mem_pair` and finish with `grind` (handles the transitivity/case-split). `Pairwise (onFun Disjoint ![...])`: `intro i j hij; rw [Function.onFun, disjoint_iff, eq_empty_iff_forall_notMem]; intro x hx; fin_cases i <;> fin_cases j <;> simp_all [...]`.
- **Bridging custom `Set` ‚Üî Mathlib `_root_.Set Object`**: `coe_inj'` (@[simp]), `coe_subset`, `coe_intersection`/`coe_empty`/`coe_union` rewrite between the two. For `coe_ssubset`: `rw [ssubset_iff_subset_ne (Mathlib side), SetTheory.Set.ssubset_def (custom side), coe_subset]; simp [coe_inj']` ‚Äî the custom `‚äÇ` has NO Preorder instance so `ssubset_iff_subset_ne` only fires on the Mathlib coe side; use the custom `ssubset_def` (`X‚äÇY ‚Üî X‚äÜY ‚àß X‚âÝY`) for the other. For `coe_Disjoint`: `rw [Set.disjoint_iff_inter_eq_empty, ‚Üê coe_intersection, ‚Üê coe_empty, coe_inj', SetTheory.Set.disjoint_iff]` (qualify `disjoint_iff` ‚Äî it's ambiguous with Mathlib's).
- **`singleton_iff` pattern** ("no proper nonempty subset ‚Üî singleton"): forward ‚Äî take `x` from `nonempty_def`, show `A={x}` by `ext`; for `y‚ààA` use `by_contra`, then exhibit `{x} ‚äÇ A` (proper since `y‚ààA, y‚àâ{x}`) and nonempty to contradict the hypothesis. Backward ‚Äî from `B ‚äÇ {x}` and `B‚âÝ‚àÖ`, pull `b‚ààB`, show `B={x}` (every element forced to `x` via `hsub`+`mem_singleton`, and `x=b‚ààB`), contradicting strictness.
- **Specification from replacement**: to get `‚àÉB‚äÜA, ‚àÄx, x.val‚ààB ‚Üî P x` without `specify`, use `A.replace (P := fun (a:A) (y:Object) => y = a.val ‚àß P a)` with functionality `hP` proved by `hy.trans hy'.symm`; membership via `replacement_axiom`, and collapse `a.val=x.val ‚Üí a=x` with `Subtype.coe_injective`.
- **Chapter 3 SetTheory.Set existence results**: when given `h : axiom_of_universal_specification` (= `‚àÄ P, ‚àÉ A, ‚àÄ x, x‚ààA ‚Üî P x`), every "‚àÉ set with membership ‚Üî œÜ" exercise is just `obtain ‚ü®X, hX‚ü© := h (fun z => œÜ z); exact ‚ü®X, hX‚ü©` (emptyset/singleton/pair/union/specify/replace all this shape; for emptyset use `fun _ => False` and `(hX x).mp hx`). `univ_iff` backward: build `U.specify (fun u => P u.val)` and finish with `specification_axiom' (A := U) _ ‚ü®x, hU x‚ü©` + `simpa`. `no_univ := by rw [‚Üê univ_iff]; exact Russells_paradox`.
- **Regularity (`not_mem_self`/`not_mem_mem`)**: apply `axiom_of_regularity (nonempty_of_inhabited hmem)` to a singleton `{(A:Object)}` / pair `{A,B}`; `axiom_of_regularity` yields `‚ü®x, hx‚ü©` with `hx : ‚àÄ S:Set, x.val = ‚ÜëS ‚Üí Disjoint S _`. Identify `x.val` via `(mem_singleton _ _).mp x.property` / `(mem_pair _ _ _).mp x.property`, get `Disjoint`, then `rw [disjoint_iff, eq_empty_iff_forall_notMem]` and feed `(mem_inter _ _ _).mpr ‚ü®_, _‚ü©` to derive the contradiction. `mem_inter`/`mem_singleton`/`mem_pair` all take explicit Object args.
- **Decidable counterexamples over custom `Function` (Chapter 3)**: `apply isFalse; intro H` then apply `H` to concrete witnesses. Build functions with `mk_fn (fun x => ‚Ä¶)`; evaluate with `eval_of` (`(mk_fn f) x = f x`) and `comp_eval` (`(g‚óãf) x = g (f x)`). Use `Nat` as the generic infinite set; `(0:Nat)`/`(1:Nat)` are distinct by `simp` (via `ofNat_inj`). For "g‚àòf injective but g not": pick domain `X = ({(0:Object)}:Set)` (a singleton) so injectivity is vacuous ‚Äî any two elements equal via `Subtype.ext` of `(mem_singleton _ _).mp x.property`. For "g‚àòf onto but f not": pick codomain `Z = ({(0:Object)}:Set)` so `onto` is trivial. NOTE: `g (f x)` where `g = mk_fn (fun _ => c)` reduces with a SINGLE `eval_of` (the inner `f x` is discarded) ‚Äî don't double-rewrite. `mk_fn`/`mem_singleton` need the `SetTheory.Set.` prefix in the `Chapter3` namespace.
- **Gluing functions on a union (`Function.glue`/`glue'`)**: build the Mathlib function `hfun : (X‚à™Y) ‚Üí Z := fun w => if hw : w.val ‚àà X then f ‚ü®w.val,hw‚ü© else g ‚ü®w.val, (hmemXY w).resolve_left hw‚ü©` (with `classical`), then `refine ‚ü®mk_fn hfun, ‚ü®?_,?_‚ü©, ?_‚ü©`. Key lemma to prove once: `hincl_val : (inclusion h a).val = a.val` by `simp [Function.inclusion, eval_of]`. CRITICAL: `(inclusion h a).val` is NOT defeq to `a.val` (to_fn uses choice), so you can't `exact`/`rw` through it ‚Äî and `rw [hincl_val ‚Ä¶]` on the if-SCRUTINEE fails with "motive is not type correct". Instead: after `comp_eval, eval_of`, set `hv := hincl_val ‚Ä¶`, then discharge the `dif_pos`/`dif_neg` condition with `(by rw [hv]; exact ‚Ä¶)` and fix the function argument with `congr 1; apply Subtype.ext; exact hv`. To swap an argument before applying a hypothesis (e.g. `hfg` on the intersection), `rw [show (‚ü®(inclusion ..).val, h‚ü© : X) = ‚ü®y.val, h'‚ü© from Subtype.ext hv]`. Uniqueness: `by_cases hwX : w.val ‚àà X` (NOT `rcases hmemXY` ‚Äî the Y-case mustn't assume ‚àâX), build `inclusion .. ‚ü®w.val,_‚ü© = w` via `Subtype.ext (by rw [hincl_val ..])`, then `calc` through `comp_eval` + the hypothesis.
- **Custom `Function` (Chapter 3, SetTheory.Set)**: `Function.eq_iff f g : f = g ‚Üî ‚àÄ x, f x = g x` (use `rw [eq_iff]; intro x`). `comp_eval` unfolds `(g ‚óã f) x = g (f x)`. `inverse_eval h y x : x = (f.inverse h) y ‚Üî f x = y`. Inverse facts: `inverse_comp_self h x : (f.inverse h)(f x)=x`, `self_comp_inverse h y : f ((f.inverse h) y)=y`. one_to_one via `rw [one_to_one_iff]` then push hyp through `f` with `by rw [hyp]` + `rwa [self_comp_inverse, self_comp_inverse]`.
- Defeq `if h:P then ... else ...` with a known proof: `rw [dif_pos hex]` substitutes your `hex` for the chooser; then `congr 1` + `Nat.cast_injective hex.choose_spec` to pin the witness.

## Conventions
- Proofs intentionally follow the textbook structure (comments say so); prefer faithful over golfed.
- Chapter namespaces: `Chapter9`, `Chapter10`, etc. Custom `Nat` lives in Section 2.
